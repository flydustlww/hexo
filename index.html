JavaScript面向对象精要--读书笔记

##基本类型和引用类型

###1.1什么是类型
原始类型保存为简单的数据值，引用类型保存为对象，其本质是指向内存位置的引用。
为了让开发者能够把原始类型和引用类型按照相同的方式处理（为啥要用相同的方式？）js花费了很大的力气来保证语言的一致性。
其他编程语言用stack存储原始类型，用heap存储引用类型。
js则不同：它用一个变量对象追踪变量的生命周期（原始值被直接保存在变量对象内，而引用值则作为一个指针保存在变量对象内，也就是说大家都保存在变量对象内）

###1.2原始类型
boolean,number,string,null,undefined
js和其他语言一样，原始类型的变量直接存储原始值（引用类型存的是内存中的存储位置）

####鉴别原始类型
typeof ‘hfskjdhf’ // string
typeof 10 // number
typeof true // boolean
typeof undefined // undefined
typeof null // object empty object point
use value === null // true or false to
console.log(“5” == 5); // true
console.log(“5” === 5); // false
== 会转换类型 ===则不会

####原始方法
虽然string,number,boolean是原始类型值，但是他们也有各自的方法(null undefined没有)
‘dfjslf’.toLowerCase();
10.toFixed(2);
true.toString();
‘fhsdkf’.charAt(0)

###1.3引用类型
引用类型指js中的对象，同时也是在js中最接近类的东西。
引用值是引用类型的实例，（请说人话？）
对象是属性的无序列表，属性包含key（key必须是sting）和value

es6 const let var区别

##1
var 变量提升。var声明的变量可以在声明之前使用，值为undifined
let 变量不提升，存在暂时性死区()。let声明的变量只绑定当前区域，不再受外部影响。zai 在代码块内的声明之前都是不可使用的。

虽然tmp已经用var提前声明，但是作用域内又用let声明了一个局部变量tmp，导致if内的tmp绑定在后者这个块级区域，所以在let声明前赋值，会报错。
在“暂时性死区”中，typeof不再是一个100%安全的操作。

##2
let 不允许重复声明

##为什么需要let（为什么需要块级作用域）？
1 es5只有全局作用域和函数作用域，如果没有块级作用域，会导致函数作用域的变量覆盖全局作用域的变量
2 for循环计数的变量，在循环结束后会泄漏为全局变量
3 块级作用域使得IIFE（立即执行匿名函数）不再必要

const 声明的是只读常量。且声明就要复制，不能留到以后赋值。
与let相同的是，const声明的变量只在所在的块级作用域内有效，而且不可以重复声明
const声明的变量并不是变量的值不可改动，而是变量所指向的内存地址不可改动。对于简单类型的（数值 字符串 布尔值）而言，值就保存在内存地址中。而对于数组和对象这种复合类型的的变量而言，变量指向的内存地址保存的是指针，const只能保证这个指针不能变，至于指针所指向的数据结构是不是可以变化，const不能保证。
